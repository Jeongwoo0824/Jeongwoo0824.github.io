  • 스미싱 탐지 시스템

    - 주요 업무 : Backend - 기술 스택 : Windows / Node.js, MySQL - 업무 기간 : 2022.03 ~ 2022.9 (약 6개월)

    코로나로 인한 방역지침이나 확진자추이 등을 미끼로 SMS를 통한 신종 사기 수법이 증가하고 있습니다. 정보취약계층의 보호와 범죄피해를 방지하기 위해 이 어플리케이션을 개발하게 되었습니다. SMS의 경우 AI를 통해, URL의 경우 바이러스 토탈 API를 통해 스미싱 여부를 판단합니다. 백엔드 개발언어로는 Node.js를 기반으로한 Express프레임워크를 이용하여 개발 편의성을 높이고 RestAPI를 구현하였습니다. 데이터베이스는 MySql을 사용하고 단일 데이터 베이스를 이용했습니다.

     #JWT 토큰을 이용한 계정 로그인 인증

     POSTMAN 과 JWT(Json Web Token)을 사용하여 토큰 인증 방식의 로그인을 구현하였습니다.  JWT의 단점 중 하나인 토큰탈취를 방지하기 위해 Access Token의 만료시간을 단축시켰으며  짧아진 만료시간은 Refresh Token을 사용하여 보완하였습니다.

     #신고 기능 및 신고 누적 시 블랙 리스트 기능 구현

     REST API 와 MySQL 트리거를 사용한 스미싱 신고 및 블랙리스트 기능을 구현하였습니다.  사용자가 스미싱 의심 SMS 신고 시 MySQL 트리거를 통해 신고 누적 수를 확인 후  1일 이내 10건 이상 누적 시 블랙리스트 등록되는 기능입니다.  블랙리스트 등록 시 앱 이용자에게 블랙리스트에 해당되는 계정정보(핸드폰 번호)로 문자가 오게되면 스미싱 경고문구가 나타납니다.  뷰를 이용하여 블랙리스트에 등록된 스미싱 의심자의 PID, 전화번호, 신고사유를 쉽게 조회 및 사용할 수 있도록 구현하였습니다.

     #회원가입, 계정 삭제, 신고 누적 시 계정 정보 업데이트

     SQL 프로시저를 사용하여 사용자가 계정 생성이나 수정 또는 신고 누적시 계정정보(핸드폰 번호)를 조회해,  새로운 계정 생성 및 삭제하며 신고 누적된 블랙리스트 계정정보를 반환합니다.  중복되는 계정이 있을 시 첫번째로 생성된 계정으로 업데이트됩니다.
